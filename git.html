Gitコマンド実用百科事典 100選

初心者からプロフェッショナルまで使える、100の主要コマンドとオプションの完全ガイド

1. 設定・ユーザー変更 (Config)

コマンド解説
git config --global user.name "Name"PC全体のユーザー名を登録
git config --global user.email "Email"PC全体のメールアドレスを登録
git config --local user.name "Name"【リポジトリ単位】現在のプロジェクトのみ名前変更
git config --local user.email "Email"【リポジトリ単位】現在のプロジェクトのみメール変更
git config --list適用されている全設定を表示
git config --show-origin user.name設定がどこ（ファイル）で定義されているか確認
git config --global core.editor "code --wait"エディタをVS Codeに変更
git config --global alias.st statusgit stでstatusを呼べるようにエイリアス作成
git config --global core.quotepath false日本語ファイル名の文字化けを防止
git config --global init.defaultBranch mainデフォルトブランチ名をmainに固定

2. リポジトリの開始・取得

コマンド解説
git init現在のディレクトリをGit管理下にする
git clone [url]リモートリポジトリを丸ごとダウンロード
git clone --depth 1 [url]履歴を捨てて最新の1コミット分だけ軽量クローン
git clone --branch [name] [url]特定のブランチを指定してクローン
git sparse-checkout巨大リポジトリの一部ディレクトリだけを抽出して取得

3. 基本操作 (Add/Commit)

コマンド解説
git add [file]特定のファイルをステージング
git add .変更があった全てのファイルをステージング
git add -u追跡済みのファイルの変更・削除のみをステージング
git add -p変更箇所（ハンク）を対話形式で確認しながらadd
git commit -m "msg"メッセージを付けてコミット
git commit -am "msg"addとcommitを同時に行う(新規ファイルは対象外)
git commit --amend直前のコミットメッセージや内容を修正
git commit --amend --no-editメッセージ変更なしで直前のコミットに含める
git commit --allow-empty -m "msg"変更がなくても空のコミットを作成
git mv [old] [new]ファイル名を変更してGitに通知
git rm [file]ファイルを削除してステージング
git rm --cached [file]ファイルは残すがGitの管理対象から外す

4. 差分・状態確認 (Diff/Status)

コマンド解説
git status現在のステージング・変更状態を表示
git status -sbブランチ情報を含め、短い形式で表示
git diff【差分】ワークツリーとステージングの差
git diff --cached【差分】ステージングと最新コミットの差(commit直前確認)
git diff HEADワークツリーと最新コミットの差
git diff [commit1] [commit2]コミットID間の差分を比較
git diff [branch1]..[branch2]ブランチ間の差分を比較
git diff --name-only変更があったファイル名の一覧だけ表示
git diff --word-diff行単位ではなく、単語単位で差分を表示
git diff --statファイルごとの変更行数の統計を表示

5. 履歴表示 (Log/Show)

コマンド解説
git log詳細なコミット履歴を表示
git log --oneline履歴を1行ずつ簡潔に表示
git log -pコミット履歴と一緒に実際の変更内容（差分）も表示
git log --graph --all --onelineブランチの分岐をアスキーアートで表示
git log -n 5最新の5件だけ表示
git log --author="Name"特定の作者のコミットだけ抽出
git log --grep="Fix"コミットメッセージの内容で検索
git log --since="2 weeks ago"最近2週間のコミットを表示
git log [file]特定のファイルに関する変更履歴のみ表示
git log -S "code"特定のコードが含まれるようになったコミットを検索
git show [commit_id]特定のコミットの詳細内容を表示
git shortlog -sn各貢献者のコミット数をランキング表示

6. ブランチ・切り替え

コマンド解説
git branchローカルブランチ一覧を表示
git branch -aリモートを含む全ブランチを表示
git branch [name]新しいブランチを作成
git switch [name]ブランチを切り替える
git switch -c [name]作成と切り替えを同時に行う
git branch -d [name]マージ済みのブランチを安全に削除
git branch -D [name]マージに関係なく強制削除
git branch -m [new_name]現在のブランチの名前を変更
git branch --mergedマージが完了しているブランチを表示
git branch --no-mergedまだマージされていないブランチを表示
git checkout [commit_id]一時的に過去のコミット状態に移動
git switch -1つ前にいたブランチに素早く戻る

7. リモート操作

コマンド解説
git remote -vリモートリポジトリのURLを確認
git remote add origin [url]リモートリポジトリを登録
git fetch originリモートの最新情報を取得（合流はしない）
git pull origin [branch]リモートから取得してマージ
git push origin [branch]ローカルの変更を送信
git push -u origin [branch]次回からgit pushだけで済むよう上流設定
git push origin --delete [branch]リモート側のブランチを削除
git remote prune originリモートで消されたブランチの参照を掃除
git remote set-url origin [url]リモートの接続先URLを変更
git fetch --all --prune全リモートの最新情報を取得し、不要な枝を掃除

8. 取消・復元 (Reset/Restore)

コマンド解説
git restore [file]ワークツリーの変更を破棄して元に戻す
git restore --staged [file]addしたファイルを取り消す（ステージ解除）
git reset --soft HEAD^直前のコミットのみ取消（変更はステージに残る）
git reset --mixed HEAD^直前のコミットとaddを取消（変更は残る）
git reset --hard HEAD^直前のコミットと変更をすべて完全に破棄
git reset --hard [commit_id]特定の過去コミットまで全てを強制的に戻す
git revert [commit_id]過去のコミットを打ち消す「逆コミット」を作成
git checkout HEAD [file]特定のファイルを最新コミットの状態に戻す
git reflogreset等で消したコミットを含む全操作履歴を表示
git reset --hard HEAD@{1}reflogを使い、間違えて行ったresetを取り消す
git cherry-pick [commit_id]他のブランチから特定のコミットだけをコピーして持ってくる
git cherry-pick --abortチェリーピックで競合が起きたので中止する

9. 一時退避・タグ・掃除

コマンド解説
git stash作業途中の変更を一時的に隠す
git stash list退避させた作業一覧を表示
git stash pop最後に隠した作業を復元してリストから消す
git stash apply復元するがリストには残す
git stash drop特定の退避データを削除
git clean -dfGit管理外の不要なファイルやフォルダを削除
git tag [tag_name]現在のコミットに軽量タグを付ける
git tag -a [v1.0] -m "msg"注釈付きタグ（署名等を含む）を付ける

10. 高度な操作・メンテナンス

コマンド解説
git rebase [branch]ブランチの基点を移動して履歴を一直線にする
git rebase -i HEAD~3過去3つのコミットをまとめたり修正したりする
git merge --no-ff [branch]必ずマージコミットを作って合流する
git blame [file]ファイルの各行を「誰がいつ」書いたか表示
git grep "keyword"リポジトリ内のファイルを高速検索
git bisect startバグが混入したコミットを二分探索で特定する
git archive --format=zip HEAD現在のファイルをzip形式で書き出す
git gc不要なデータを最適化・圧縮（ガベージコレクション）
git fsckデータベースの整合性をチェック
